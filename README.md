# 🎬 React Movie App (Каталог фильмов)

Полноценное веб-приложение для поиска фильмов, просмотра трейлеров и изучения фильмографии актеров. Проект использует публичное API TMDB (The Movie Database), но запросы проксируются через собственный Node.js сервер для обеспечения безопасности API-ключей.

![Главная страница приложения](image.png)

## ✨ Основной функционал

* **🔍 Умный поиск и Фильтрация:** Поиск фильмов по названию и фильтрация по жанрам. При клике на жанр в любом месте приложения происходит автоматический переход на главную с примененным фильтром.
* **🎞️ Детальная информация:** Просмотр описания фильма, рейтинга и встроенных YouTube-трейлеров (через безопасный домен `youtube-nocookie`).
* **🎭 Навигация по актерам:** Интерактивный список каста (горизонтальный скролл). При клике на актера открывается его персональная страница с полной фильмографией.
* **❤️ Избранное (Закладки):** Возможность сохранять любимые фильмы. Данные управляются через Redux Toolkit и синхронизируются с `localStorage` браузера (ничего не пропадает при обновлении страницы).
* **⏳ UX/UI Улучшения:** Плавные Skeleton-загрузчики вместо скучных текстов «Загрузка...», кастомные скроллбары, обработка ошибок и страница 404 (Not Found).
* **🔄 Пагинация:** Бесконечная подгрузка списка фильмов по кнопке «Показать еще».

## 🛠️ Стек технологий

**Frontend:**
* **React 18** (Functional Components, Hooks)
* **React Router v6** (Динамический роутинг, передача state при навигации)
* **Redux Toolkit (RTK)** (Управление стейтом приложения, слайс для Избранного)
* **RTK Query** (Кэширование API-запросов, управление состояниями загрузки `isLoading`/`isFetching`)
* **CSS3** (Flexbox, CSS Animations для скелетонов, Custom Scrollbars)

**Backend:**
* **Node.js & Express** (Создание прокси-сервера)
* **Axios** (Выполнение запросов к TMDB API)
* **Dotenv** (Скрытие секретных API-ключей)
* **CORS** (Настройка кросс-доменных запросов)

## 🧠 Архитектурные решения
* **Безопасность API:** Клиент (React) никогда не обращается к TMDB напрямую. Все запросы идут через `http://localhost:5001/api/...`, что позволяет скрыть ключ авторизации на сервере.
* **Оптимизация запросов:** Использование параметра `append_to_response` в TMDB API позволяет за один запрос получать данные о фильме, его трейлерах и актерах, сокращая нагрузку на сеть.
* **Умная навигация:** Использование хуков `useNavigate` и `useLocation` для проброса ID жанра между изолированными компонентами (например, из карточки фильма на страницу актера, а оттуда — на главную).
